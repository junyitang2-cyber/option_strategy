# Option Strategy — 教学模式升级设计规格

**Date**: 2026-05-25
**Status**: Phase 1 Complete / Phase 2-3 Pending
**Topic**: 教学模式优化 — 加深 Greeks 理解 + 策略构造认知 + 小白友好

---

## Context

当前教学模式的核心问题：

1. **Greeks 只有数字没有解释** — 6 个希腊值各只有一行注释，对所有策略一样
2. **损益图不解释"形状为什么这样"** — 只显示组合总和，没有逐腿分解
3. **没有学习路径** — 用户可以从新手级跳到专家级，无引导无警示
4. **图表是静态的** — 无 tooltip、无区域着色、无时间衰减动画、无概率可视化
5. **Playbook 模板化严重** — chartUse 对所有 55+ 策略完全一样

目标用户：知道 Call/Put 基本概念，但不理解 Greeks、波动率、策略构造逻辑。

## Part 1: Greeks 概念卡片系统

### 1.1 触发方式

每个希腊值数字旁边加 `[?]` 图标，点击展开浮层概念卡片。

### 1.2 卡片结构（三层）

```
┌─ Delta 概念卡片 ────────────────────────────┐
│                                              │
│  📖 是什么？                                  │
│  Delta 衡量期权价格对标的股价变化的敏感度。     │
│  Call Delta ∈ (0, 1), Put Delta ∈ (-1, 0)    │
│  ATM 期权 Delta ≈ ±0.5                        │
│                                              │
│  🎯 你当前的策略: Iron Condor                 │
│  组合 Delta = +0.03                           │
│  → 接近中性。标的涨 $1，组合仅赚约 $3          │
│  → Iron Condor 不赌方向，赚的是时间价值        │
│                                              │
│  💡 怎么看？                                  │
│  • |Delta| > 0.5: 强方向性策略                │
│  • |Delta| < 0.1: 市场中性的收租/波动率策略    │
│  • 负 Delta: 偏空头，适合看跌                  │
│                                              │
│  🔗 相关概念: Gamma, Delta Neutral            │
└──────────────────────────────────────────────┘
```

### 1.3 六个希腊值的教学要点

| 希腊值 | 概念核心 | 策略解读逻辑 |
|--------|---------|-------------|
| **Delta** | 方向暴露；ATM≈±0.5 | 组合 Delta 解读（方向性强 / 中性 / 空头） |
| **Gamma** | Delta 的变化率；ATM 近到期时峰值 | 正 Gamma=凸性收益（买方），负 Gamma=怕跳空（卖方） |
| **Theta** | 每日时间损耗；卖方正 Theta；近到期加速 | 正 Theta=时间朋友，负 Theta=每天亏钱 |
| **Vega** | IV 敏感度；远到期高 Vega | 高 Vega=赌 IV 变化，低 Vega=不关心波动率 |
| **Rho** | 利率敏感度；远到期+深度 ITM 时才有意义 | 当前低利率环境 → 基本可忽略 |
| **Risk/PnL** | 不单独展开卡片，改到损益图区域 | - |

### 1.4 实现

- 扩展 `greekPanels` 数组，每个加 `concept`、`interpretation`（策略解读模板）、`benchmark`（判断标准）
- 浮层组件用绝对定位，不遮挡图表
- "相关概念" 可点击跳转到对应希腊值卡片
- 卡片内容根据当前策略动态生成策略解读部分

## Part 2: 策略构造可视化（逐腿分解）

### 2.1 切换按钮

损益图上方加三个标签切换：
```
[组合视图] [逐腿分解] [概率锥]
```

### 2.2 逐腿分解模式

- 每条腿用不同颜色绘制独立 PnL 曲线
- 组合曲线用白色加粗叠加在最上层
- 图例自动展开，每条腿标注颜色 + 描述（如 "Short 1 CALL @ 105"）
- 腿编辑器卡片加同色左边框标记
- 每条腿可单独开关显示（checkbox），默认全开
- 悬停曲线时高亮该腿

### 2.3 教学价值

- Iron Condor 的利润平台 = Short Put + Short Call 两个卖方腿重叠区域
- Butterfly 中间 Short 2 腿如何"挖出"利润尖峰
- Calendar 近月和远月腿如何互相抵消又保留时间差

### 2.4 实现

- `curveData()` 增加 `perLeg: true` 选项，返回每条腿的独立曲线数据
- `chartSvg()` 支持多条曲线 + 腿颜色映射
- 颜色方案：固定 6 种颜色循环，保证相邻腿颜色可区分

## Part 3: 学习路径 + 进度引导

### 3.1 策略列表加难度标签

每个策略列表项加难度颜色圆点：
- 🟢 新手  🟡 中级  🟠 高级  🔴 专家  🔵 向导

增加难度筛选按钮：`[新手] [中级] [高级] [专家]`

### 3.2 学习路径推荐条

选定策略后，在详情区顶部显示：

```
← 前置: Bull Put Spread    当前: Iron Condor    下一步: Butterfly →
         (中级)                                  (高级 · 理解比例价差)
```

推荐逻辑：
- 前置 = 同家族中难度更低的策略
- 后续 = 同家族中难度更高的，或概念相近的其他家族策略
- 每个推荐带具体原因说明（替代现有 `relatedReason()` 的类别套话）

### 3.3 学习进度追踪（localStorage）

```javascript
{
  "visited": ["long-call", "bull-call-spread", "iron-condor"],
  "completed": ["long-call", "bull-call-spread"],
  "currentLevel": "intermediate"
}
```

- 策略卡片上可点击 `✓ 已理解` 标记完成
- 进度条：当前难度层级内已完成/总数
- 完全离线

### 3.4 难度跃升警告

当用户从未完成过中级策略、直接点击专家级策略时，弹出提示：
- 提醒该策略需要的前置知识
- 建议先学的策略列表
- 两个按钮：`[仍然查看]` `[先去学前置策略]`

触发条件：目标策略难度 - 当前最高已完成难度 >= 2 级

## Part 4: 图表交互增强

### 4.1 Hover Tooltip

- SVG `mousemove` 事件，根据鼠标 X 坐标反推现货价格
- Tooltip 显示：现货价 + 到期 P&L + 当前 P&L + 开仓 P&L
- 跟随鼠标移动，触摸设备用 `touchmove`

### 4.2 盈亏区域着色

- 到期 P&L > 0 的区域用半透明绿色填充
- 到期 P&L < 0 的区域用半透明红色填充
- 零线加粗

### 4.3 盈亏平衡线标注

盈亏平衡点旁边直接标注数值，如 `$94.60`、`$105.40`

### 4.4 时间衰减播放按钮

```
[▶ 播放] [⏸ 暂停] [↺ 重置]  速度: [1x] [2x] [5x]
```

- 播放时 DTE 从当前值自动递增到 max DTE
- 图表实时更新，直观看到 Theta 侵蚀
- 特别适合对比 Long Call（时间敌人）vs Iron Condor（时间朋友）

### 4.5 概率锥叠加

- 用已有 `normCdf()` 计算 ±1σ / ±2σ 范围
- 在到期损益图上叠加半透明正态分布
- 标注"到期时股价约有 XX% 概率落在盈利区"

## Part 5: Playbook 教育内容重写

### 5.1 chartUse 策略特异性

替代当前所有策略相同的 chartUse 文本：

```
Long Call → "看绿色到期线从哪个现货价开始转正（盈亏平衡点）；注意曲线弯曲度反映 Gamma 加速效果"
Iron Condor → "看利润平台宽度（两短腿之间）；注意两侧坡度反映长腿保护的成本"
Butterfly → "看利润尖峰的位置和宽度；尖峰越窄盈利区间越小但成本也越低"
Calendar → "近月和远月两条到期线的差异；时间推进后近月腿衰减更快"
```

### 5.2 setup/entry 去模板化

不再按 `textTone()` 分类填充，改为：
- 从策略自身的 `money`/`risk`/`when` 字段提取关键信息驱动
- 每段都带具体的策略名称、数值、场景

### 5.3 pitfalls 增强

在现有机械规则（short腿→分配风险、Theta负→时间敌人、Vega高→IV风险）基础上，加入策略特定的常见错误：

```
Long Call → "买太虚值的 Call：虽然便宜但胜率极低"
Iron Condor → "腿间距太窄导致盈亏比失衡"
Straddle → "忽略 IV Crush：即使方向对了也可能亏钱"
Butterfly → "腿间距不对称导致意外方向暴露"
```
